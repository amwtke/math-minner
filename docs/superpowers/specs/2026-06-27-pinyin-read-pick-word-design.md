# 拼音玩法五：「🔎 看拼音选词」设计文档

**日期**：2026-06-27
**作者**：amwtke（设计协作：Claude）
**状态**：设计已确认，待写实现计划

## 1. 目标与约束

在现有「语文·拼音」学科里新增**第 5 个玩法**：读出屏幕上的拼音（一个字或一个词的拼音），从 **4 个候选汉字/词**里点出对应的那个。本质是「👂 听音找字」的**看字版**——把"听发音"换成"看拼音"，逼孩子真正去**读拼音**（尤其声调），而不是凭耳朵或熟脸认字。

硬约束（沿用现有基因，不得破坏）：

- **复用 CNQ 共用壳**：和「听音找字 / 声调 / 拼音拼读」一样跑在 `#cnq` 屏的 `CNQ` 运行时上，新增一个 `READ` 对象即可，不新建屏。
- **共享体系**：矿石、星星 ★、心、镐子/皮肤、错题强化（`Weak`）、题库选择（`gamePool`）全部复用；存档同一份。
- **运行时零第三方依赖、离线可用**：不引入任何新数据文件、新依赖。干扰项只用现有数据里的读音信息（声母 `sm`/韵母 `ym`/声调）。
- **存档对 `server.py` 透明**：仅在 `S` 上加一个关卡进度字段。

非目标（YAGNI，本期不做）：

- **不建形近字对照表**——已选"读音相近"路线。
- **不做多音字专门机制**——真正的多音字需要每个读音单独的拼音+音频数据，属于被否掉的数据活，留作以后增强。
- 候选数固定 4 个，不做首关 3 个的特例（如手感不好再调）。

## 2. 玩法与入口

- **入口**：在「选择玩法」`renderModes('cn')` 里加**第 5 张卡**，配色 `--red`（当前仅世界 7 卡背景用到，可作模式卡主色），标题 `🔎 看拼音选词`，副标题"读出拼音，选对的字 / 词"，`onclick="READ.start()"`。
- **运行屏**：复用 `#cnq` 壳——
  - `qTitle`：`🔎 看拼音选词 · 第 N 关`
  - `qPrompt`：放大号的**带调拼音**（如 `yú` 或 `lǎo hǔ`），下方一行 `qHint`"读出拼音，选对应的字/词"
  - `qBody`：4 个候选按钮，复用 `listenGrid` + `cnqChar` 样式（汉字大字）

## 3. 出题与难度曲线（字 → 词 递进）

在 `gamePool(kind, stage)`（index.html:1458）新增 `kind==='read'` 分支：

- **默认题库**：
  - 单字池 = 世界 `1..min(3+stage,6)` 的单字（复用 `_cnPool(_cnMaxWorld(stage))`，沿用现有由易到难、逐步放复韵母/鼻韵母的节奏）。
  - **第 4 关（`stage>=3`）起**把词语（世界 7–8）并入题池；关卡越高，目标抽到词语的比例越高。
- **自主命题 / 跟着课本**：直接用 `S.qsource.items` 里录入的字 + 词（`gamePool` 已有的 custom 分支风格）。
- **候选数**：固定 4。池子太小的小题库自动收缩到池大小（`n = min(4, pool.length)`）；可用候选不足 2 时 `toast` 提示"换题库或玩法"并 `G.goModes('cn')`，与现有玩法的兜底一致。

目标选取沿用 `Weak.pick(pool, last)`：偏向错题/相似题，避免与上一题连续重复。

## 4. 干扰项算法（核心）

**铁律：3 个干扰项的"带调拼音"都不得等于正确项的拼音。** 否则同一段拼音对应两个候选，题目有歧义（如 鱼/渔 都是 `yú`）。判等用每项的 `PD(h).py`（带调）比较。

### 4.1 单字目标（正确字 T，拼音 `pyT`）

1. **候选来源**：当前题池（默认题库下优先同世界、再扩大到整池；与 `LISTEN.pick` 的"同世界优先"一致）。
2. **排除**：T 本身；以及任何 `PD(h).py === pyT` 的字（去歧义）。
3. **择优"读音易混"**：优先 **同韵母** 或 **同声母**（复用 `Weak.similar(T, pool)` 的同 `sm`/同 `ym` 思路）。其中最佳是 **同声母+同韵母、只差声调**（如 `yú`→雨 `yǔ`、语 `yǔ`；`shì`→十 `shí`、史 `shǐ`）——专练声调与平翘舌 / 前后鼻。
4. **补齐**：易混项不够 `n-1` 个时，用池内随机字补足（仍受排除规则约束）。
5. 把正确项与干扰项 `shuffle` 后渲染。

### 4.2 词语目标（正确词 T）

1. **候选来源**：题池中的**其他词语**。
2. **排除**：T 本身；`PD(h).py === PD(T).py` 者。
3. **择优**：同世界 → 同结构（叠词 / 同字数）→ 与 T 共享一个汉字；不足再用其他词补齐。
4. `shuffle` 后渲染。

> 说明：48 个词语里几乎找不到"读音相近"的词，所以词语难度来自"同结构、需读准整串拼音才能区分"（如 `bà ba`/`mā ma`/`gē ge`）。

## 5. 音频策略（这是"读"的游戏，不是"听"的）

- **答题前不发声**——否则退化成「听音找字」，违背"通过拼音找词"的初衷。
- **答对后**：高亮选中项 + 朗读该字/词（`playPy(PD(target).code)`），作为奖励与音—形—义强化。
- **答错**：`beep('bad')`、扣心、标红（`cnqMiss`）、`Weak.miss(target)`；心为 0 进失败遮罩。

## 6. `READ` 对象骨架（紧挨 `PM` 放，index.html:1749 之后）

与 `LISTEN` 同构：

- `start()`：`S.subject='cn'`；`stage=S.readStage`；`pool=gamePool('read',stage)`；池不足兜底 toast；建 `CNQ={kind:'read', hearts, maxHearts, score:0, cleared:0, combo:0, goal:6+stage, n:4, depth:min(1+floor(stage/2),4), busy:false, pool, cur:null}`；`show('cnq')`；`next()`。
- `next()`：`Weak.pick` 选目标 → 判定字/词 → `pick(target)` 造 4 选项 → `qPrompt` 显示 `PD(target).py` → 渲染候选按钮、绑 `answer`。**不**自动播音。
- `pick(target)`：实现 §4 的干扰项算法（按字 / 词分支）。
- `answer(hz, el)`：对则 `cnqHit`+`playPy`+`Weak.hit`+`cnqGood`+`cnqHUD`，达 `goal` → `win()` 否则 `next()`；错则 `Weak.miss`+`cnqMiss`，心尽 → `lose()`。
- `win()` / `lose()`：`cnqWin('readStage', ()=>READ.start(), '下一关开始出现词语，读准每个音节！')` / `cnqLose(()=>READ.start())`。

## 8. 验收（手动，跑 `python3 server.py` 点测）

- **默认题库**：早关全单字；任一题 4 个选项的拼音互不与正确项相同，且明显在练声调易混（同声母韵母差声调）；**第 4 关起**出现词语，干扰为同结构词。
- **音频**：答题前静默；答对朗读、答错扣心、心尽进失败遮罩；过关 `readStage++`、给星和宝箱。
- **存档**：过几关后换设备 / 重登同名，`readStage` 续上。
- **小题库**（自主命题 < 4 字）：候选自动收缩到池大小；< 2 时友好 `toast` 并退回玩法屏。
- 项目无前端单测框架，按上面清单人工点测即可（可借 `verify` / chrome-devtools 技能驱动浏览器）。
