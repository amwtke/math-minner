# 英语矿工 设计文档

**日期**：2026-06-29
**作者**：amwtke（设计协作：Claude）
**状态**：设计待确认（本文档供审阅）

## 1. 目标与约束

在现有「方块数学矿工」里新增第三个学科 **📕 英语**，给**小学六年级（基础水平：认词 + 简单句）**练英语，重点是**英语语感**而非死记单词。第一期覆盖玩家选定的四种语感：**听力 / 语序句型 / 词汇搭配 / 拼写自然拼读**。

硬约束（沿用全项目设计基因，不得破坏）：

- **同一体系、共享积分**：矿石（木/石/铁/钻）、星星 ★、镐子+技能、皮肤、心，跨学科共享；存档同一份。
- **像素 / Minecraft 风格**：复用现有 `.tile` / `.world` / HUD / overlay 视觉与音效。
- **运行时零第三方依赖、游玩设备永远离线**：前端是静态文件，断网可玩；`server.py` **仅用标准库**。第三方工具（构建期 TTS、pypinyin 等）只在**构建/生成期**用，绝不进运行时。
- **本地多人服务器版**：进度按玩家名存 `data/<名字>.json`，新字段对 `server.py` 透明。
- **与拼音结构对齐**：玩法数量、题库模式、错题本、语速开关、win/lose overlay 全部沿用拼音那一套交互范式，孩子零学习成本。

非目标（YAGNI，本期不做）：

- 不做账号/排行榜/联机；不做手写或语音输入（孩子说、机器判）。
- 不做语法讲解/课本单元解锁（**自由主题，不绑课本**）。
- 不做整段阅读理解、完形长文（基础水平，留待后期）。
- 不做 Service Worker / PWA（保持「单文件前端」洁癖）。

## 2. 导航架构（在拼音改造之上再加一张学科卡）

现状：标题 → `#subjects`（两张卡：➗ 数学、📖 语文拼音）→ `#modes`（按 `S.subject` 动态渲染，`renderModes` index.html:1382）。语文走 `subjects → #qsrc 选题库 → modes`；数学走 `subjects → modes`。

英语**完全套用语文那条路径**：

```
标题 → #subjects（+【新】📕 英语卡）→ #qsrc（英语版题库屏：主题默认 / 自主命题）→ #modes（renderModes('en') 渲染 5 张玩法卡）
```

1. `#subjects` 增加第三张卡 **📕 英语**（复用 `.world` 大卡样式），`onclick` 进英语题库流程（首次进英语先触发音频就绪检查，见 §6）。
2. `S.subject` 取值扩展为 `'math' | 'cn' | 'en'`。`G.goModes('en')` → `renderModes('en')`。
3. `renderModes(subj)`（1382）增加 `subj==='en'` 分支，渲染 5 张英语玩法卡（见 §4）。
4. **题库屏 `#qsrc` 改为学科感知**：英语只显示**主题默认**与**自主命题**两个入口（**不显示「跟着课本」**）。题库提示文案/范围按 `S.subject` 取对应题库状态（见 §7 的 per-subject `qsource`）。
5. 返回键：英语各玩法「🏠 玩法」→ `G.goModes('en')`；`#modes` 的 `G.modesBack`（1349）已按 `S.subject` 分支，补 `'en'` → 回英语题库屏。

⚠️ 风险点：`renderModes`、`gamePool`、`modesBack`、题库屏渲染都按 `S.subject` 分支，新增 `'en'` 必须每处补齐，漏一处就串味（英语进了拼音的池子/界面）。

## 3. 英语主题地图（自由主题，按难度由易到难解锁）

不绑课本，按**儿童高频生活主题**排难度（每个主题 = 一个「世界/关组」，类比 `PINYIN_WORLDS`）。基础水平：以**具体名词**起步（emoji 友好），动作/形容词其次，短句最后。

| # | 世界 | 主题 | 例词（节选） | 引入句子？ |
|---|---|---|---|---|
| 1 | 🎨 颜色数字洞 | colors & numbers | red, blue, one, ten | 否 |
| 2 | 🐾 动物草原 | animals | cat, dog, bird, fish | 否 |
| 3 | 🍎 食物果园 | food & fruit | apple, rice, egg, milk | 短句起步 |
| 4 | 👕 身体衣物窟 | body & clothes | hand, eye, hat, shoe | 短句 |
| 5 | 🏫 学校矿井 | school | book, pen, desk, bag | 短句 |
| 6 | 🏃 日常动作谷 | daily verbs | run, eat, read, like | 短句 |
| 7 | ☀️ 天气季节崖 | weather & season | sunny, rain, hot, spring | 短句 |
| 8 | ⚽ 兴趣运动渊 | hobbies & sports | swim, sing, ball, bike | 短句（毕业关综合） |

- 每关只引入少量新词、控制 4–8 题；句子只用**已解锁**的词，**绝不超纲**。
- 句子保持**短而简单（3–5 词）**：`I like apples.` / `She is happy.` / `It is a cat.`（契合基础水平）。
- 主题清单与词量后续可加，加词只需往构建脚本词库塞、重跑一次（见 §5）。

## 4. 五种玩法（覆盖四种语感，全部复用现有引擎）

所有玩法共用：心（命）、星星 ★、矿石、镐子等级/技能、皮肤、连击、发矿与扣血计分、win/lose overlay、语速开关、错题本 `Weak`（错过的词/句加权重出；「相似题」= 同主题或易混项）。

| 玩法 | 语感 | 复用引擎 | 期 |
|---|---|---|---|
| ⛏️ 词义配对挖矿 | 词汇 | `PG` 配对（1828，近乎照搬） | 一 |
| 🎨 看图选词 | 词汇·语感 | `READ` 四选一（1760） | 二 |
| 👂 听音选词 | 听力 | `LISTEN`（1594） | 二 |
| 🔤 拼单词 | 拼写/自然拼读 | 新 `EA` 有序拼装引擎 | 三 |
| 🧩 连词成句 | 语序/句型 | 新 `EA` 有序拼装引擎 | 三 |

### 4.1 ⛏️ 词义配对挖矿 —— 泛化 `PG` 配对引擎（投入最小）

玩法：**中文 ↔ 英文** 配对（主模式），如 `苹果` ↔ `apple`，配对成功挖矿。

- **「图」是可选皮肤，不是唯一**：题库条目若带 emoji，可切「emoji ↔ 英文」模式（emoji 主要服务 §4.2 看图选词）；**默认就是中文 ↔ 英文**。
- **点方块即读音**：点任意方块（中文或英文）→ `playEn(该词)` 读出英文发音，边配边磨耳朵。（拼音版是"配对成功才读音"；英语基础阶段默认改为"点击即读"以强化「见词 + 听音」；保留一个开关可切回"答对才读"。）
- 引擎：复用 `PG.buildPinyinBoard / tap / success / fail / giveOre / 技能 / HUD`；核心配对只认 `pid` 与 `k`('q'/'a')，与内容无关，故近乎零改动。把题库生成段产出 `pairs=[{txt:'苹果', ans:'apple', audio:'apple'}]`。
- 渲染：给英文 tile 加 `.tile.en`（拉丁印刷体、字号适配），中文 tile 用 `.tile.cn`。`d.dots` 强制 `false`（防点阵计数渲染出错）。

### 4.2 🎨 看图选词 —— 复用 `READ` 四选一引擎

玩法：上方给 emoji（如 `🍎`），下方 4 个英文词选一选 `apple`；选对挖矿。**用图不用中文翻译 = 更纯的语感**。

- 抽象词或无合适 emoji → 退化为「看义选词」（显示中文义，4 选 1 英文）。
- 干扰项：同主题词 / 易混词（拼写或发音相近）。选对/选错都 `playEn`，强化音义绑定。
- 引擎：复用 `READ` 的四选一外壳（1760），把"读拼音"的题面换成 emoji/中文，发音换 `playEn`。

### 4.3 👂 听音选词 —— 复用 `LISTEN` 引擎（音频驱动）

玩法：顶部 🔊 `playEn(目标词)`，下方 N 个英文词 tile，点出听到的那个；答对复播、挖矿。

- 引擎：复用 `LISTEN`（1594），`playPy → playEn`，题池取英语 `gamePool('en_listen', stage)`。干扰项同主题/近音。
- 池子不足兜底：照 `LISTEN` 现有 `toast('…不足，换玩法或题库')` 风格（1598）。

### 4.4 🔤 拼单词 —— 新 `EA` 有序拼装引擎

玩法：听词 `playEn` + 看 emoji（如 `🍎`），下方打乱的字母块 `p a l e p`，按序点成 `a-p-p-l-e`；点错可退格；拼对挖矿+复读。

- 自然拼读友好：字母块含目标词字母 + 少量干扰字母（按难度递增）。
- 引擎：**新写轻量 `EA`（English Assembly），不改 `PM`**（`PM` 是拼音声母/韵母/声调三槽模型，与"N 个 token 有序排列"不同）。`EA` **只抄关卡外壳**：`cfg(stage)` 难度曲线、hearts/score/combo、`giveOre`、win/lose；核心交互全新（候选池 → 有序追加/退格 → 比对目标）。

### 4.5 🧩 连词成句 —— 复用 `EA`（token 换成单词）

玩法：打乱的单词块 `like / I / apples` → 按语序点成 `I like apples.`；成功 `playEn(整句)` 读出。**练语序神器**。

- 同一 `EA` 引擎，token 粒度从「字母」换成「单词」，目标从「单词」换成「短句」；基础水平句子 3–5 词。
- 干扰项：多给 1–2 个不该用的词（按难度）。标点（句号/问号）作为固定尾块或自动补。
- 句子音频：整句一个 mp3（见 §6），非逐词拼接。

> **本期不做**（留作后期，覆盖度已够四种语感）：✍️ 选词填空（完形味，搭配/时态语感）、🔗 搭配消消乐、🔀 找别扭。设计上都能复用 `READ`/`M`，按需再加。

## 5. 内容与数据生成

- **产物 = `en-data.js`**（入 git，纯文本）：`window.EN_DATA = { worlds:[...], ... }` + 必要的辅助表。
  - ⚠️ **必须是 `.js` 不是 `.json`**：给 `.json` 开静态白名单会顺带让人 GET `data/<玩家>.json` 拿别人存档；`.js` 已在白名单，安全（与拼音同理）。
- **每条词**：`{ en:'apple', zh:'苹果', emoji:'🍎', audio:'apple', theme:'food', level:3 }`。
- **每条句**（仅含句子的世界）：`{ en:'I like apples.', zh:'我喜欢苹果。', tokens:['I','like','apples'], audio:'i_like_apples' }`。
- **构建脚本 `tools/gen_en_data.py`**（仅开发机跑一次，产物入 git）：
  1. 读人工维护的主题词库/句库（`tools/en-words.txt` 之类，按主题+难度分组）。
  2. 生成 `en-data.js`（结构化 + 计算 `audio` key）。
  3. 驱动**构建期 TTS** 为**全部默认词/句**生成 `audio/en/<key>.mp3`（默认包，见 §6）。
- **audio key 规则（前后端必须一致、可独立计算）**：小写 → 非 `[a-z0-9]` 转 `_` → 合并连续 `_` → 去首尾 `_`。例：`"ice cream"→"ice_cream"`、`"I like apples."→"i_like_apples"`。无需哈希、无需映射表，前端 `playEn` 与服务器生成端各自算同一 key。
- **emoji 来源**：纯文本字符，零素材、离线、像素页友好；无合适 emoji 的词留空 `emoji`，玩法自动退化为中文义。
- **自主命题输入**：复用拼音 `#qsrc` 自主命题输入框范式；用户录英文词（可附中文义，或留空只练形音）/短句，存入 `S.enQsource.items`（见 §7）。录入时触发服务器现生成音频（见 §6.2）。

## 6. 音频系统（与拼音根本不同：英语是开放集，不能靠有限音节库覆盖）

**架构 = 预生成默认包 + 自主命题现生成**（已确认）。游玩设备永远离线；只有服务器在"生成新音频"时短暂联网。

### 6.1 默认包：构建期 TTS 预生成

- `gen_en_data.py` 在**开发机**用高质量 TTS（推荐自然、亲和的 en-US 童声/女声）一次性生成全部默认主题词/句的 `audio/en/<key>.mp3`。
- **落地子决策（待你拍板，文档先给推荐）**：默认包怎么到玩家服务器？
  - **(推荐) 直接提交进 git**：英语默认包自生成、版权干净、体量适中（数百词 + 短句，约数 MB）。好处：开箱即用、默认体验**零首次联网**，比拼音「首次联网拉取」更稳。需把 `audio/en/`（默认包）从 `.gitignore` 的 `audio/` 规则里**豁免**（如 `.gitignore` 写 `audio/` 但 `!audio/en/`，或默认包放 `audio/en/_pack/`）。
  - **(备选) 服务器首次拉取**：默认包托管一处，首次进英语时服务器 `urllib` 拉一次（完全照搬拼音 §6.1 的 `/api/audio/status`+`fetch`+`.ready` 范式）。代价：需要托管位置。
- 单字玩法直接播 `<key>.mp3`；句子玩法播整句 `<key>.mp3`（**整句一个文件**，非逐词拼接——这点和拼音逐音节拼接不同）。

### 6.2 自主命题：服务器按需现生成 + 缓存

- 玩家录入自定义词/句 → 前端 `POST /api/en/tts {texts:[...]}` → **服务器**用标准库 `urllib` 调一个免费云 TTS（HTTP 可达；具体提供方在实现期定）→ 存 `audio/en/<key>.mp3` → 返回就绪。**server 仍纯标准库（仅 urllib）**，与拼音用 urllib 拉 zip 同一手法。
- **离线兜底**：服务器无网/TTS 失败 → 不阻断游戏，静音降级（玩法纯视觉可玩），提示"服务器联网后重进即可补全发音"。与拼音离线兜底一致。
- **缓存/幂等**：已存在 `<key>.mp3` 跳过；并发用服务器锁防重复生成（沿用拼音并发保护思路）。
- 默认包同样走这套 key/缓存，自定义与默认共用 `audio/en/` 与 `playEn`。

### 6.3 前端播放 `playEn`（纯 HTML5 Audio，离线，无库）

- 仿现有 `playPy`（index.html:632）：`playEn(key)` 播 `/audio/en/<key>.mp3`，单例池 `Map` 复用，`currentTime=0` 重播，`.play().catch(()=>{})` 兜底；缺文件静默降级。
- **复用语速档位 `VOICE_SPEEDS`**（632 上方）：`playbackRate` 调速。英语多为单文件，无多音节 `gap` 拼接（句子已是整段录音）。
- **iOS 首次须手势解锁**：复用现有 `_unlockAudio`（点学科卡/首次点击触发），所有 `playEn` 都在点击回调内（tap/success/题面播放均在手势链）。
- `server.py` 静态白名单已含 `.mp3`（拼音已加），**无需再改 MIME**；新增 `/api/en/tts` 端点（~40 行，`urllib` 标准库）。`audio/en/` 与 `audio/pinyin/` 平级，现有静态服务自动托管。

## 7. 状态与存档

新增 state（`freshState` index.html:592 初始化）：

- `S.subject` 扩展支持 `'en'`。
- 英语各玩法关卡（类比拼音的 `matchStage/listenStage/...`，用 `en` 前缀避免与拼音串档）：
  `enMatchStage, enPickStage, enListenStage, enSpellStage, enSentStage`（均初值 0）。
- **题库 per-subject**：现 `S.qsource`（599）被拼音占用；英语用独立 `S.enQsource = {type:'default', label:'默认主题', items:null}`，英语版 `gamePool` 与题库屏只读/写它，**不碰拼音 `qsource`**（防跨学科串题库）。

存档：

- `SAVE_FIELDS`（656）追加 `'enMatchStage','enPickStage','enListenStage','enSpellStage','enSentStage','enQsource'`。
- `applySave`（716 一带）照现有兜底：英语 stage 缺键补 0；`S.enQsource` 缺/坏则重置默认（防"第 NaN 关"）。
- 运行时引擎对象（`PG` 复用、`EA` 新增等）用局部持有，**不进存档**（类比 `CR`）。
- 经济系统（res/tool/skin/stars）天然跨学科共享，无需新增。`server.py` 的 `_post_save` 原样存任意 JSON，**不改 Python 存档逻辑**。

## 8. 分期实现

- **一期（闭环）**：`#subjects` 加英语卡 + 导航/题库屏学科化（含 `gamePool('en_*')` + `S.enQsource`）+ ⛏️ 词义配对挖矿（泛化 `PG`，中英配对 + 点击发音）+ `gen_en_data.py` 内容管线 + `en-data.js` + 音频接入（默认包预生成 + `playEn` + `/api/en/tts` + 自主命题现生成）。跑通 登录 → 选英语 →（音频就绪）→ 配对 → 挖矿 → 升镐 全闭环。
- **二期**：🎨 看图选词（复用 `READ`）+ 👂 听音选词（复用 `LISTEN`）。两个轻量复用，吃一期搭好的音频与样式。
- **三期**：🔤 拼单词 + 🧩 连词成句（新 `EA` 有序拼装引擎 + 句子音频）。覆盖拼写与语序两种语感。
- **后期（按需）**：✍️ 选词填空、🔗 搭配消消乐、🔀 找别扭。

## 9. 风险与对策

| 风险 | 对策 |
|---|---|
| `S.subject` 三态后，`renderModes/gamePool/modesBack/题库屏` 漏补 `'en'` 分支 → 串味 | 逐处枚举补齐 + 回归测试数学/语文两学科入口与返回键 |
| 跨学科题库串档（拼音自定义题库泄漏到英语） | 英语用独立 `S.enQsource`，与拼音 `qsource` 物理隔离 |

| 英文/中文混排在像素英文字体下乱 | 新增 `.tile.en`（拉丁印刷体）、`.tile.cn` 已有；字号适配 |
| 自主命题任意词/句无预生成音频 | 服务器 `urllib` 现生成 + 缓存；离线则静音降级 + 提示，不阻断 |

## 11. 运行时零依赖确认

构建期 TTS / 词库脚本只在开发机跑，产出纯静态 `en-data.js`（入 git）+ 默认 mp3 包。运行时：前端纯 HTML5 Audio 播放本地 mp3；`server.py` 仅在自主命题现生成时用标准库 `urllib` 调云 TTS（与拼音用 urllib 拉 zip 同级），其余全程零第三方依赖、游玩设备 100% 离线。数学与语文玩法完全不受影响。
